Build a robust, production-ready Homepage Slider Management System that allows administrators to fully control homepage slider content through a dedicated Admin Panel, while delivering a seamless, high-performance visual experience to end users on the frontend. The system must be clean, image-only, fully responsive, and optimized for speed and scalability.

🎯 Objectives & Goals:

The system must achieve the following core goals with no compromise:

Slides are entirely managed through the Admin Panel — no hardcoding of content in templates
Each slide displays images only — absolutely no text overlays, headings, descriptions, buttons, captions, or UI elements on top of the slide image
The slider must render fully responsive and pixel-perfect across all screen sizes including mobile, tablet, and large desktop (4K-ready preferred)
The system must prioritize performance, accessibility, and maintainability
🛠️ Admin Panel Requirements:

1. Navigation & Menu Structure

Add a dedicated section in the Admin Panel sidebar or navigation menu labeled "Homepage Slider" under the Content Management group. This item should be accessible only to users with appropriate roles (e.g., Super Admin, Content Manager).

2. Slide Management Features

The admin interface must provide the following capabilities in a clean, intuitive UI:

✅ Add New Slide — A form or modal to upload and configure a new slide
✅ Upload Image — Supports JPG, PNG, and WebP formats only; all other formats must be rejected with a clear error message
✅ Set Display Order — Drag-and-drop reordering interface is strongly preferred; fallback to numeric input field if drag-and-drop is not feasible
✅ Enable / Disable Slide — Toggle switch per slide to control visibility without deleting the record
✅ Delete Slide — With a confirmation dialog to prevent accidental deletion
✅ Preview Slide Image — Inline thumbnail preview in the management table, with an option to open a full-size lightbox preview
3. Image Validation & Processing Rules

All uploaded images must comply with the following rules, enforced at both the client side (for UX) and server side (for security):

Recommended dimensions: 1920×700 pixels (landscape, desktop-optimized)
Mobile scaling: Images must scale responsively using CSS (object-fit: cover) — no cropping distortion
Maximum file size: 2MB hard limit per upload; display a user-friendly error if exceeded
Image optimization: Automatically compress and optimize images server-side before saving (e.g., using Sharp, ImageMagick, or equivalent)
WebP conversion: If the uploaded image is not already WebP, auto-convert it to WebP and store both formats (original as fallback)
Unique filename generation: Rename uploaded files using UUID or timestamp-based naming to prevent conflicts and path traversal vulnerabilities
4. Admin UI/UX Standards

Display all slides in a sortable data table with columns for: Thumbnail, File Name, Status (Active/Inactive), Display Order, Date Added, and Actions
Show total active/inactive slide counts at the top of the management page
Provide success/error flash messages for all CRUD operations
Implement input validation with clear inline error feedback
💾 Database Structure:

Create the following table to store slider data. All fields are required unless stated otherwise:

Table name: homepage_slides

Column	Type	Description
id	INT, Primary Key, Auto Increment	Unique identifier
image_url	VARCHAR(500)	Relative or absolute path to the stored image
image_webp_url	VARCHAR(500), Nullable	Path to the WebP-converted version of the image
is_active	BOOLEAN, Default: TRUE	Controls whether the slide is shown on the frontend
display_order	INT, Default: 0	Controls the rendering sequence of slides
created_at	TIMESTAMP	Record creation timestamp
updated_at	TIMESTAMP	Last modification timestamp
Indexing: Add a composite index on (is_active, display_order) to optimize frontend query performance.

🌐 Frontend Slider Requirements:

1. Core Slider Behavior

Auto-advance: Slides transition automatically every 4–5 seconds (make interval configurable)
Transition style: Smooth fade or slide transition — implement whichever renders more cleanly at full width; avoid jarring or choppy animations
Infinite loop: After the last slide, seamlessly return to the first slide with no visual break
Pause on hover: On desktop/pointer devices, auto-advance pauses when the user hovers over the slider and resumes on mouse leave
Touch/Swipe support: On mobile and tablet, support natural left/right swipe gestures to manually navigate between slides
Keyboard accessibility: Support left/right arrow keys for navigation when the slider is focused
2. Strict Content Rules (Non-Negotiable)

The following elements must never appear on any slide:

❌ Text overlays of any kind
❌ Call-to-action buttons or links
❌ Captions, titles, or descriptions
❌ Logo or branding overlays
✔️ Only the full-width image, rendered edge-to-edge with no padding or margin
3. Navigation Controls (Optional but Recommended)

Dot indicators: Display subtle dot/pill indicators at the bottom of the slider to show slide position and allow direct navigation
Arrow buttons: Optional — if implemented, use minimal, semi-transparent arrows that do not distract from the image; hide arrows on mobile
If no navigation controls are desired, omit entirely for a fully clean experience
