PERFORMANCE OPTIMIZATION BRIEF FOR PAKCART.STORE

CRITICAL ISSUES TO ADDRESS:

1. REDUCE FIRST CONTENTFUL PAINT (FCP) FROM 6.6s TO <1.8s:
   - Implement critical CSS inlining in the <head> tag
   - Defer non-critical JavaScript files using async/defer attributes
   - Minimize render-blocking resources (CSS and JavaScript)
   - Preload key fonts used above the fold using <link rel="preload">
   - Remove unused CSS from above-the-fold content
   - Consider using a CDN for static assets to reduce latency
   - Implement image lazy loading to prevent blocking page renders

2. IMPROVE LARGEST CONTENTFUL PAINT (LCP) FROM 9.2s TO <2.5s:
   - Optimize and compress all images used in hero sections
   - Convert images to modern formats (WebP) with fallbacks
   - Implement responsive images with srcset attributes
   - Enable GZIP compression for text-based assets
   - Minimize server response time (TTFB - Time to First Byte)
   - Cache static content aggressively on client and server
   - Use image CDN (like Cloudinary or Imgix) for on-the-fly optimization
   - Preload the LCP image in the <head> section

3. REDUCE TOTAL BLOCKING TIME (TBT) FROM 950ms TO <200ms:
   - Break up long JavaScript execution tasks using setTimeout/requestIdleCallback
   - Minify and compress JavaScript files
   - Remove or defer unused JavaScript libraries
   - Implement code splitting to load JS only when needed
   - Profile JavaScript using Chrome DevTools to identify heavy computations
   - Consider using web workers for heavy calculations
   - Reduce third-party script execution time (analytics, ads, tracking)
   - Use performance.mark() and performance.measure() to identify bottlenecks

4. DECREASE SPEED INDEX FROM 10.1s TO <3.6s:
   - Implement critical rendering path optimization
   - Serve fonts with font-display: swap to prevent text rendering delays
   - Minimize CSS and JavaScript files
   - Load above-the-fold content first
   - Use HTTP/2 Server Push for critical resources
   - Implement service workers for offline support and resource caching
   - Use Intersection Observer API for lazy loading non-critical content
   - Prioritize visible content rendering over below-the-fold content

5. GENERAL OPTIMIZATIONS:
   - Enable Brotli compression (better than GZIP)
   - Implement a Content Delivery Network (CDN) globally
   - Use a service worker strategy for network efficiency
   - Minimize DOM size and complexity
   - Remove unused dependencies from package.json
   - Implement code splitting for route-based or component-based chunks
   - Enable caching headers (Cache-Control: max-age=31536000 for immutable assets)
   - Audit and remove render-blocking resources
   - Use critical CSS extraction tools
   - Monitor Core Web Vitals continuously with real user monitoring (RUM)
